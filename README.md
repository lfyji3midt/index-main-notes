# index-main-notes

## 项目简介

这是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为若干静态 HTML 页面的集中存放与版本管理空间。每个页面可视为独立的笔记、工具、演示或实验性内容。

## 目录说明

```
/
├── index.html          # 仓库首页（可选，用于展示页面列表或导航）
├── pages/              # 存放各个独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 页面引用的静态资源（图片、样式、脚本等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：核心目录，每个 HTML 文件对应一个独立页面。
- `assets/`：存放页面所需的公共或私有资源文件，按类型分目录管理。

## 页面归档说明

本仓库中的 HTML 页面均为独立文件，彼此之间无强关联。每个页面可以：

- 自包含样式与脚本（内联或引用 `assets/` 中的资源）
- 使用简单的 HTML 结构，方便直接打开查看
- 通过 GitHub Pages 或其他静态托管服务发布

新增页面时，请将 HTML 文件放入 `pages/` 目录，并确保资源引用路径正确。

## 维护说明

- **添加页面**：新建 HTML 文件到 `pages/`，如需资源请放入 `assets/` 对应子目录。
- **更新页面**：直接修改对应 HTML 文件，注意保持与其他页面的独立性。
- **删除页面**：移除 `pages/` 中的文件以及 `assets/` 中不再使用的资源。
- **版本控制**：建议每次修改后提交清晰的 commit 信息，描述变更内容。

## 使用方式

1. 克隆本仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件即可预览。
3. 若需部署，可将仓库设为 GitHub Pages 源，或上传至任意静态文件服务器。
